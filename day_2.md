AI SECURITY, ETHICS & USAGE GUIDELINES

1.  Five Security Risks When Using AI Coding Tools

Data Leakage: Sensitive code or secrets (API keys, credentials) may
be exposed when pasted into AI tools.

Training Data Exposure: AI tools may store or learn from prompts,
risking unintended reuse of proprietary code.

Malicious Code Generation: AI may generate insecure or vulnerable
code (e.g., SQL injection risks).

Dependency Risks: AI may suggest outdated or vulnerable libraries.

Prompt Injection Attacks: Malicious inputs can manipulate AI tools
to leak or override instructions.

2.  Code Ownership Issues with AI-Generated Code

- Ownership may be unclear depending on the AI tool’s terms of
  service.
- Generated code may resemble copyrighted training data.
- Legal risks if code unintentionally copies existing proprietary
  work.
- Organizations must define clear policies on AI-generated code usage.
- Always review licensing and compliance requirements.

3.  AI Usage Guidelines (1-Page Document)

Purpose: Ensure safe, ethical, and effective use of AI tools in
development.

Guidelines:

- Do not input sensitive data (API keys, passwords, PII).
- Always review and validate AI-generated code.
- Use AI as an assistant, not a decision-maker.
- Follow company coding and security standards.
- Verify dependencies and libraries suggested by AI.
- Document AI usage in critical code sections if required.
- Avoid blind copy-pasting without understanding.

4.  Appropriate vs Inappropriate AI Use

Appropriate: 1. Generating boilerplate code 2. Explaining concepts 3.
Debugging assistance 4. Writing test cases 5. Refactoring suggestions

Inappropriate: 1. Handling sensitive data (credentials, PII) 2. Making
critical security decisions without review 3. Copying code without
understanding 4. Using AI in restricted or regulated environments
without approval 5. Relying on AI for legal/compliance decisions

5.  PII Handling in AI Prompts

PII (Personally Identifiable Information) includes: - Names, emails,
phone numbers - Addresses, IDs, financial details

Best Practices: - Never include real user data in prompts - Use
anonymized or dummy data - Mask sensitive fields - Follow data
protection policies

6.  When to Prioritize Learning Over AI Assistance

Use AI: - When stuck after trying - For quick explanations - To speed up
repetitive tasks

Prioritize Learning: - When learning new concepts - During interviews or
exams - When building core fundamentals - When debugging critical logic

Rule: If you cannot explain the solution yourself, you should not rely
fully on AI.
